21 /* USAGE DOCUMENTATION
22 General
23 --------
24 The CmdLine class is used to specify the options a program or class wishes
25 to handle, and then process the {argc,argv} arguments of a standard main
26 call.
27 
28 Parameter types
29 ---------------
30 There are 2 types of parameter switches - standalone and parameterized.
31  A standalone switch takes no values, eg -h or -help
32  A parameterized switch takes a value, eg -s J:in
33 
34 Since the switch is just a string value, you are not restricted to the <dash><letter>
35 syntax, eg -h. For example, --help or ==help or **help are all acceptable.
36 
37 Setup
38 -----
39 The user decides which parameter types and string values to represent them.
40 The member functions addStandaloneSwitch() and addParameterSwitch() are used
41 to set them. For a parameterized switch, a string initialValue can be specified
42 as a default.
43 
44 The member function process(argc,argv) is used to go over the argv array
45 to check whether the switches are set, and in the case of parameterized
46 switches, to take in the parameter value.
47 
48 Query
49 ------
50 In use, the member function switchValue() returns the values "set" or "unset"
51 for standalone switches, and the parameter value or its initial value,
52 for parameterized switches.
53 
54 The NumberOfRemainingParameters is used to detect any parameters in argv
55 that were not processed. This allows CmdLine to ignore switches it is not
56 interested in and still allows the user to process them later, via the
57 RemainingParamter(int index).
58 
59 Additional utilities
60 ---------------------
61 printSwitches and dump are self explanatory.
62 
63  */
